# LeanGuide - プロジェクト概要

## コンセプト

**「頑張っているのに痩せない原因を可視化し、次に取るべき行動を示すダイエット支援アプリ」**

LeanGuideは、単なるカロリー記録アプリではない。ダイエットに取り組んでいるのに成果が出ない人が「なぜ痩せないのか」を理解し、具体的な改善アクションを得られることを目的としたアプリである。

---

## 解決したい課題

多くのダイエッターが抱える問題：

- カロリーを記録しているだけで、何が原因で停滞しているか分からない
- PFC（タンパク質・脂質・炭水化物）のバランスを意識できていない
- 体重の日々の増減に一喜一憂してしまい、傾向を正しく捉えられない
- 運動しているつもりでも、実際の消費カロリーと摂取カロリーの関係が見えない

---

## LeanGuideのアプローチ

### 1. データから原因を分析する

直近7日間の食事・体重・運動データを統合的に分析し、停滞や増加の原因を特定する。分析は優先度付きで、最も重要な課題から順に提示される。

分析の観点：
- カロリー収支（摂取 vs 目標）
- PFCバランスの偏り
- 週末と平日の食事パターンの違い
- 運動量の過不足
- 体重トレンド（7日間移動平均による評価）

### 2. 具体的な改善アクションを提案する

「カロリーオーバーです」で終わらず、「具体的に何をすれば良いか」をアクションとして提示する。例えば：
- 「タンパク質が不足しています → 朝食に卵やヨーグルトを追加してみましょう」
- 「週末にカロリーが跳ね上がっています → 外食時は事前にメニューを確認しましょう」

### 3. 正しい評価で一喜一憂を防ぐ

体重は日単位の変動が大きいため、7日間移動平均で傾向を評価する。これにより「昨日より0.5kg増えた」ではなく「この1週間で緩やかに減少傾向にある」という正しい認識を促す。

## 主な機能

| 機能 | 概要 |
|------|------|
| ダッシュボード | カロリー円形グラフ、PFCバランス、体重/体脂肪の現在値と目標差を一画面で把握 |
| 食事記録 | 文科省 日本食品標準成分表（約2,500品目）から検索入力。お気に入り登録で繰り返し入力を効率化 |
| 体重記録 | 体重・体脂肪率の記録。推移グラフで14日間のトレンドを表示 |
| 運動記録 | 12種類のプリセット活動 + 自由入力。時間入力で消費カロリーを自動推定 |
| グラフ | 30日間のP/F/C・カロリー・体重・体脂肪率を折れ線グラフで可視化。最大2グループ同時表示 |
| 原因分析 | 7日間のデータを統合分析し、注意・情報・良好の3段階でフィードバック |
| ボディメイク設定 | 目標体重と達成期限から1日の目標カロリー・PFCを自動算出。PFCプリセット（バランス型、高タンパク型等）対応 |

## 技術構成

| 項目 | 技術 |
|------|------|
| フレームワーク | Next.js 16（App Router, Turbopack） |
| 言語 | TypeScript |
| スタイリング | Tailwind CSS 4 |
| データベース | Neon PostgreSQL + Drizzle ORM |
| 認証 | NextAuth.js v5（Google OAuth + メール/パスワード） |
| 食品データ | 文部科学省 日本食品標準成分表 2,526品目（ローカルJSON） |
| ホスティング | Vercel |
| リポジトリ | GitHub（パブリック / OSS） |

## ターゲットユーザー

- ダイエットに取り組んでいるが、思うように成果が出ていない人
- カロリー管理はしているが、PFCバランスまでは意識できていない人
- 体重の増減に振り回されがちで、正しい傾向把握をしたい人
- 日本語環境で、日本の食品データベースを使いたい人
